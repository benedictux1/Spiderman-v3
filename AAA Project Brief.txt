EXECUTIVE SUMMARY
================================================================================

Kith Platform is a sophisticated personal intelligence system designed to help users organize, analyze, and derive insights from their personal relationships and social interactions. The platform combines modern web technologies with AI-powered analysis to transform unstructured notes and conversation data into actionable relationship intelligence.

The project has undergone continuous development with recent major improvements in January 2025, resulting in:
- Streamlined contact creation with robust alerts and real-time table refreshes
- Enhanced progress UI that surfaces detailed error messages from import status
- Improved Telegram sync with fallback polling mechanisms for progress tracking
- Advanced UI components including progress bars that auto-scroll into view
- Fixed sync modal with direct event binding and proper state management
- Comprehensive error handling and user feedback systems

TECHNICAL ARCHITECTURE
================================================================================

Backend Stack:
- Framework: Python Flask 2.3.3 with SQLAlchemy 2.0.21 ORM
- Database: SQLite with WAL (Write-Ahead Logging) mode for concurrency
- AI Integration: OpenAI GPT (configurable model) for relationship analysis
- Vector Database: ChromaDB 0.4.15 for semantic search and RAG operations
- Task Scheduling: APScheduler for background processing
- External Integration: Telethon 1.34.0 for Telegram chat import

Frontend Stack:
- Core: Vanilla JavaScript (modularized architecture)
- UI: HTML5, CSS3 with responsive design
- Modules: 
  * /static/js/main.js - Core functionality, UI management, and profile operations
  * /static/js/contacts.js - Contact management & CRUD operations
  * /static/js/settings.js - Settings and admin functionality

Database Schema:
- Users: Authentication and user management
- Contacts: Core contact information with tier system (1, 2, 3)
- Raw Notes: Unstructured user input and imported data
- Synthesized Entries: AI-processed and categorized relationship data
- Import Tasks: Background task tracking with progress and status
- Contact Audit Log: Immutable audit trail of all contact changes

DEPLOYMENT & CONFIGURATION
================================================================================

Environment Setup:
- Python 3.8+ required
- Virtual environment recommended (venv/ included in project)
- Environment variables required in .env file:
  * OPENAI_API_KEY=your_openai_api_key_here
  * TELEGRAM_API_ID=your_telegram_api_id
  * TELEGRAM_API_HASH=your_telegram_api_hash
  * KITH_API_TOKEN=your_secure_api_token

Installation Process:
1. Clone repository and navigate to project directory
2. Create and activate virtual environment: python -m venv venv
3. Install dependencies: pip install -r requirements.txt
4. Configure environment variables in .env file
5. Initialize database: python app.py (first run creates tables)
6. Access application at http://localhost:5000

Production Deployment:
- render.yaml included for cloud deployment
- Gunicorn WSGI server configuration
- SQLite database with WAL mode for production use
- Logging configured for both file and console output
- Background scheduler for automated tasks

TECHNICAL SPECIFICATIONS
================================================================================

Database Schema Details:

Users Table:
- id (Primary Key), username, password_hash, created_at
- Relationships: One-to-many with contacts

Contacts Table:
- id (Primary Key), user_id (Foreign Key), full_name, tier
- Telegram fields: telegram_id, telegram_username, telegram_phone, telegram_handle
- Metadata: is_verified, is_premium, telegram_last_sync
- Timestamps: created_at, updated_at
- Relationships: One-to-many with raw_notes and synthesized_entries

Raw Notes Table:
- id (Primary Key), contact_id (Foreign Key), content, source
- Metadata: created_at, telegram_message_id, telegram_date, tags

Synthesized Entries Table:
- id (Primary Key), contact_id (Foreign Key), category, content
- Metadata: confidence_score, created_at, last_updated

Import Tasks Table:
- id (Primary Key - UUID), user_id, contact_id, task_type
- Status tracking: status, progress, status_message, error_details
- Timestamps: created_at, completed_at

Contact Audit Log Table:
- id (Primary Key), contact_id, user_id, event_timestamp
- Event tracking: event_type, source, before_state, after_state, raw_input

API Endpoints:
- GET /: Main application interface
- POST /analyze_note: Process and categorize note content
- GET /contacts: Retrieve contact list with filtering
- POST /contacts: Create new contact with validation
- PATCH /contacts/<id>: Update contact information
- DELETE /contacts/<id>: Delete contact with cascading
- POST /contacts/bulk-delete: Delete multiple contacts
- GET /contact/<id>: Get detailed contact profile
- GET /contact/<id>/raw-logs: Get audit trail for contact
- PUT /contact/<id>/categories: Replace contact categories
- GET /contact/<id>/audit-log: Get complete audit history
- POST /search: Perform semantic or keyword search
- GET /export/csv: Export comprehensive data as CSV
- POST /telegram/start-import: Initiate Telegram import with progress tracking
- GET /telegram/import-status/<task_id>: Poll import status
- POST /process-note: Manual note analysis and processing
- POST /save-synthesis: Save analyzed data with audit logging
